Agent Runtime Toolkit

给你的项目加上 Agent 能力

Keel 是一个 Python 工具包。加几行代码，你的项目就能跑 Agent Loop、管理上下文、执行工具、输出结构化数据。

$ pip install keel-runtime
5 分钟上手 浏览全部模块

Quick Start

定义工具，加个装饰器，你就有了 Agent

用 @keel.tool 定义工具，用 @keel.agent 把函数变成 Agent。不需要部署服务，不需要注册中心。

深入了解每个模块
quickstart_agent.py 
import keel_runtime as keel

# 1. 定义工具
@keel.tool(name="get_weather",
          description="Get weather for a city")
def get_weather(city: str) -> str:
    return f"{city}: 22C, sunny"

# 2. 定义 Agent
@keel.agent(
    client=MyLLMClient(),
    tools=[get_weather],
    system_prompt="You are a weather assistant.",
    max_iterations=5,
)
async def weather_agent(question: str) -> str:
    return question

# 3. 调用
result = await weather_agent("Tokyo weather?")
print(result.output)  # "Tokyo: 22C, sunny"

核心能力

Agent 系统的通用骨架，你来填领域差异

Agent Loop

LLM 调用、工具执行、迭代控制、结构化输出解析，串成完整循环。你只管写 system prompt 和 output type。

Context 管理

四分区 token 预算策略：System / Task / History / Active。自动裁剪已消费的 tool result，防止上下文爆炸。

Tool 协议

装饰器定义工具，自动生成 JSON Schema，注册和执行一体化。支持同步和异步，校验入参和出参。

AgentLoop

LLM 调用 + 工具执行 + 迭代控制

把 LLM client、context 组装、工具执行、输出解析串成完整循环。返回 status / output / iterations / 完整消息历史。

查看示例
核心
agent_loop.py
from keel_runtime.loop import AgentLoop, AgentLoopConfig
from keel_runtime.context import Message

loop = AgentLoop(
    config=AgentLoopConfig(max_iterations=10),
    chat_client=my_llm_client,
    tools=tool_registry,
    context_provider=context,
)

result = await loop.run(
    messages=[Message(role="user", content="分析这份报告")]
)
# result.status → "completed" | "tool_limit" | ...
# result.output → 最终文本输出
# result.iterations → 迭代次数
PrefixStableContext

上下文管理

token 预算、前缀稳定分区、已消费 tool result 清理、历史裁剪。四分区：SYSTEM → TASK → HISTORY → ACTIVE。

查看示例
核心
context.py
from keel_runtime.context import PrefixStableContext, ContextConfig

context = PrefixStableContext(
    config=ContextConfig(max_tokens=4096),
    system_prompt="你是一个数据分析助手",
)

# 四分区自动管理
context.set_task("请分析 Q3 销售数据")
context.add_history(Message(role="user", content="聚焦华东区"))

messages = context.assemble()  # → 完整消息列表
ToolRegistry / @tool

工具注册与执行

装饰器定义工具，自动生成参数 schema，注册和执行一体化。ToolCall 入参，ToolResult 出参。

查看示例
协议
tools.py
from keel_runtime.tools import ToolRegistry

registry = ToolRegistry()

@registry.tool(description="查询数据库")
def query_db(sql: str, limit: int = 100) -> list[dict]:
    # 自动生成 JSON Schema: {sql: str, limit: int}
    return execute(sql)[:limit]

# 执行工具调用
result = await registry.execute(ToolCall(
    name="query_db",
    arguments={"sql": "SELECT * FROM orders"}
))
# result → ToolResult(output=..., success=True)
parse_output / extract_json

结构化输出

从 LLM 文本中提取 JSON，支持 markdown code block，Pydantic 校验，纯文本 fallback。

查看示例
输出
output.py
from keel_runtime.output import parse_output, extract_json
from pydantic import BaseModel

class Analysis(BaseModel):
    summary: str
    confidence: float

llm_text = '这是分析结果 ```json\n{"summary": "增长趋势", "confidence": 0.92}\n```'

# 自动提取 JSON + Pydantic 校验
result = parse_output(llm_text, model=Analysis)
# → Analysis(summary="增长趋势", confidence=0.92)

# 纯提取，不校验
raw = extract_json(llm_text)  # → {"summary": "增长趋势", ...}
InProcessRuntime

进程内运行时

把 Python async callable 包装成 Keel job，不需要起子进程。适合单机、测试、快速验证。

查看示例
运行时
runtime.py
from keel_runtime.runtime import InProcessRuntime

async def my_agent(payload: dict) -> dict:
    # 你的 Agent 逻辑
    return {"result": "done"}

runtime = InProcessRuntime(handler=my_agent)

# 无需子进程，直接在当前进程内运行
async for event in runtime.run(job, spec):
    print(event.type, event.message)
JobManager

Job 生命周期管理

创建、运行、停止、恢复、查询、导出。支持任务依赖和 artifact 传递。

查看示例
运行时
manager.py
from keel_runtime.manager import JobManager

mgr = JobManager(store_path="./keel_data")

# 创建并运行
job = await mgr.create(spec=my_agent_spec)
await mgr.start(job.id)

# 查询状态
status = await mgr.get_status(job.id)

# 停止 / 恢复
await mgr.stop(job.id)
await mgr.resume(job.id)

# 导出产物
artifacts = await mgr.export_artifacts(job.id)
ModelConfig

模型配置

结构化模型配置，声明式 fallback，provider 校验。支持 OpenAI / Anthropic / 自定义 provider。

查看示例
配置
models.py
from keel_runtime.models import ModelConfig

config = ModelConfig(
    provider="openai",
    model="gpt-4o",
    temperature=0.3,
    max_tokens=2048,
    fallback=[
        ModelConfig(provider="anthropic", model="claude-sonnet-4-20250514"),
    ]
)

# Provider 校验 — 确保环境变量已配置
config.validate()
# 自动 fallback：主模型失败时切换到备选
AgentSpec / AgentJob

Agent 定义与状态

声明式 Agent 定义（名称、模型、资源限制），Job 跟踪运行状态和产物。

查看示例
配置
specs.py
from keel_runtime.specs import AgentSpec
from keel_runtime.jobs import AgentJob

spec = AgentSpec(
    name="data-analyst",
    model_config=my_model_config,
    description="分析业务数据并输出报告",
    max_iterations=20,
    tools=[query_db, generate_chart],
)

# Job 跟踪运行状态
job = AgentJob(spec=spec)
# job.status → created / running / succeeded / failed

events

流式事件

实时事件流：Agent 启动、工具调用、输出就绪、错误。可订阅、可过滤。

查看示例
事件
events.py
from keel_runtime.events import JobEvent, EventType

# 事件流随 Job 运行实时产出
async for event in job.events():
    match event.type:
        case EventType.STATUS:
            print(f"状态变更: {event.message}")
        case EventType.TOOL_CALL:
            print(f"工具调用: {event.data['tool']}")
        case EventType.OUTPUT:
            print(f"输出就绪")
        case EventType.ERROR:
            print(f"错误: {event.message}")
HumanGate

人工确认

独立确认原语：在 Agent 工作流的关键步骤插入人工审批节点。

查看示例
安全
gate.py
from keel_runtime.gate import HumanGate

gate = HumanGate()

# 在关键步骤请求人工确认
request = gate.create_request(
    action="发送邮件",
    reason="Agent 生成了客户邮件，需要人工确认后发送",
    payload={"to": "client@example.com", "body": "..."},
    timeout_seconds=300,
)

decision = await gate.wait_for_decision(request)
# decision.status → approved / rejected / timeout

设计原则

为什么 Keel 长这样

工具包，不是外壳

pip install，加几行代码，你的代码获得能力。不需要部署 Keel 服务，不需要注册 Agent。你写 Python，Keel 提供运行时。

通用机制，不提供领域模式

Agent Loop、Context、Tool Protocol 是通用的。Agent 怎么定义、Pipeline 怎么串，你决定。Keel 不替你做架构选择。

Provider 协议，不锁实现

ContextProvider、LLM client 都可以替换成你的实现。架构对齐 OpenAI Agents SDK 模式，但模型不锁 OpenAI。

零外部依赖

核心包不依赖任何第三方库。S3 持久化、FastAPI 集成等通过 extras 按需引入。
